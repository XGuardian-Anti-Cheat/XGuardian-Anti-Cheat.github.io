The mean-aggregated original predictions serve as input to the random forest model for match prediction. Through its decision-making process, the model generates XGuardian's final judgment. The following diagram illustrates how the sample player in the above table was determined. In this diagram, X-axis indicates the input value of the sample for the given feature; Y-axis denotes the adjusted SHAP values. For more intuitive visualization, the original SHAP values (computed by the model interpreter) are offset so that the value corresponding to the dynamic threshold probability equals 0. This value reflects the difference between the sample's predicted positive-class probability and the dynamic threshold. A transformed SHAP value of 0 indicates the prediction probability exactly matches the dynamically computed threshold on the training set, while positive/negative values represent deviations above/below the threshold; All non-case-study samples are colored dimgray; Circles represent ground-truth cheaters; Triangles represent ground-truth normal players; The case study sample is highlighted with a purple star marker. The sample's input value of 0.6722031113333333 lies above the decision boundary, causing the model to classify it as positive (i.e., a cheating player).
